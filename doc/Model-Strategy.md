# Copilot CLI 模型使用策略

## 订阅与可用模型

**学生计划订阅**

### GitHub 内置模型

| 模型 | 高级请求倍率 | 说明 |
|------|-------------|------|
| GPT-5.4 mini | 1x | 消耗高级请求额度 |
| GPT-5.3-Codex | 1x | 消耗高级请求额度，代码优化 |
| GPT-5.2-Codex | 1x | 消耗高级请求额度，代码优化 |
| GPT-5.4-mini | 0.33x | 消耗较少额度 |
| GPT-5-mini | 0x | 免费，不消耗额度 |
| GPT-4.1 | 0x | 免费，不消耗额度 |

### BYOK API（自有密钥）

| 模型 | 说明 |
|------|------|
| DeepSeek-V3.2 | 性价比高，代码能力强 |
| Qwen3.6-plus | 中文能力强，长文本处理好 |

---

## 任务类型与模型分配

### 1. 科研文献调研
**推荐：Qwen3.6-plus (BYOK)**
- 中文理解能力强，适合中文文献调研
- 长文本处理好，可以处理大量文献
- BYOK 成本低

### 2. 科研计划制定
**推荐：GPT-5.4 mini / GPT-5.3-Codex (1x)**
- 需要强逻辑推理和规划能力
- 消耗 GitHub 额度但值得
- 复杂任务用更好的模型

### 3. 代码撰写
**推荐：GPT-5.3-Codex / GPT-5.2-Codex (1x) 或 DeepSeek-V3.2 (BYOK)**
- Codex 系列专门为代码优化
- DeepSeek 代码能力也不错，性价比高，适合大量代码生成
- 简单代码可以用 GPT-5-mini (0x)

### 4. 代码审核
**推荐：GPT-5.4 mini (0.33x) 或 GPT-5-mini (0x)**
- Code-review agent 可以配合中等模型使用
- 不需要最强的模型，0x 的 GPT-5-mini 足够

### 5. 代码迭代
**推荐：根据复杂度选择**
- **复杂 bug 修复/重构**：GPT-5.2-Codex (1x)
- **小修改/日常迭代**：GPT-5-mini (0x) 或 DeepSeek-V3.2 (BYOK)

---

## 额度用完后的策略

当 GitHub 高级请求用完后：

| 用途 | 模型 |
|------|------|
| 主力日常编码、简单问答 | GPT-5-mini (0x) |
| 需要稍强能力时 | GPT-4.1 (0x) |
| 继续不受限使用 | BYOK API (DeepSeek / Qwen) |

---

## 总体分配表

| 任务类型 | 首选 | 备选（额度用完） |
|---------|------|----------------|
| 文献调研 | Qwen3.6-plus | Qwen3.6-plus (不受影响) |
| 计划制定 | GPT-5.4 mini | GPT-4.1 (0x) |
| 代码撰写 | GPT-5.3-Codex | DeepSeek-V3.2 |
| 代码审核 | GPT-5-mini (0x) | GPT-5-mini (0x) |
| 代码迭代 | GPT-5.2-Codex | GPT-5-mini / DeepSeek |

---

## 核心原则

1. 把 GitHub 额度留给**代码相关的复杂任务**
2. 文献调研等**中文/长文本任务**走 BYOK（Qwen）
3. **日常迭代和小修改**尽量用 0x 模型或 BYOK
4. 深度代码任务优先用 Codex 系列
