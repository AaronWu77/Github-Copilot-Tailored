# Copilot Instructions Template

这个模板用于新仓库初始化时的 `copilot-instructions.md` 起始版本。

## Plan 模式约束

1. 进入 plan 模式后，任何方案都不能直接定稿。
2. 每当形成一个新的计划步骤、设计决策或执行方案时，必须先调用 `ask_user` 逐项询问用户是否接受该方案。
3. 只有在当前步骤获得用户明确确认后，才能进入下一个计划步骤。
4. 所有计划细节都确认完成后，除了更新会话内的计划文件外，还必须在仓库根目录创建 `/doc/plan.md`，并把最终计划写入该文件供用户审阅。

## `/doc/plan.md` 固定结构

`/doc/plan.md` 必须按下面顺序组织内容：

1. **功能目的**：先说明这个计划要解决什么问题、目标是什么、范围边界是什么。
2. **TodoList**：列出需要执行的任务项，便于用户快速审阅整体工作拆分。
3. **具体执行方案**：说明每个阶段准备如何实施、涉及哪些模块、预期如何落地。

## Autopilot 模式约束

1. 进入 autopilot 模式后，不得直接开始执行。
2. 每次准备开始实施前，必须先调用 `ask_user`，明确询问用户是否按照当前仓库中的 `/doc/plan.md` 执行。
3. 只有在用户明确同意后，才能继续实现、修改代码、运行命令或推进任务。
4. 如果 `/doc/plan.md` 不存在、内容过期，或用户明确表示要调整方案，必须先回到计划确认流程，再继续执行。

## 推荐补充项

在复制本模板到新仓库后，建议继续补充：

- 项目目标与非目标
- 关键目录结构
- 技术栈与版本约束
- 测试、构建、发布要求
- 不可触碰的数据、安全或部署边界
